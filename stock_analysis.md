## Problem Statement

A股散户被量化交易收割，核心原因不是信息差，而是认知差。量化赚的是速度，散户亏的是情绪。一段行情由少数几个主导逻辑驱动（交易逻辑），噪音不改方向。散户的关键问题是：

1. 当前这段行情的主导逻辑是什么？
2. 这个逻辑的强度在增强还是减弱？
3. 什么时候出现了逻辑切换的信号？
4. 看不懂逻辑的时候不盲做。

## Demand Evidence

- User has manually backtested this framework and confirmed it works in hindsight
- User trades A-shares personally and has experienced the pain of being shaken out by volatility because the underlying regime logic was unclear
- Concrete example: CPO (光通信模块)板块 — 行情的底层逻辑是AI硬件需求，个股因研发进度/市占率不同呈阶梯式涨跌但服从板块大逻辑
- Concrete example: 4月美伊战争期间 — 行情交易的是"战争是否会扩大"，中间谈判消息导致忽上忽下，但核心矛盾（关键问题不可让步）→ 震荡市 → 高抛低吸
- Pain point is real: "逻辑没看懂的盲目交易，这个是需要避免的"

## Status Quo

Current workflow: Manual reading of news, reports, forums. Slow, incomplete, no systematic identification or tracking of regime logic. All three costs are felt simultaneously:

1. Missing entry — sees the logic shift too late after the big move
2. Selling on noise — can't distinguish regime reversal from quant noise during volatility
3. Information overload — too much news/data, can't synthesize the ONE dominant logic

## Sector-First Principle

个股是从属于板块的。逻辑在板块层，个股在逻辑层。

```
板块逻辑层 (Sector Logic)
  → 例如: CPO = AI硬件算力需求逻辑
    → 个股A（龙头）：研发进度快 + 市占大 → 领涨
    → 个股B（跟风）：研发慢 + 市占小 → 跟涨但不强
    → 个股C（落伍）：技术落后 → 板块涨它不涨
```

"站在风口猪都能飞" + "覆巢之下无完卵" = 板块有风个股才有机会。

## Constraints

1. A股是主要交易标的（可扩展到港股/美股）
2. 不需要在变盘T=0立即发现 — 右侧确认后可以，但要快于散户共识
3. 系统需要支持左侧交易（捕捉前兆信号）和右侧交易（趋势确立后快速识别）
4. 核心目的是防止"看不懂逻辑的盲做" — guardrails first, edge second
5. 多数据源并行采集，AI综合合成
6. 板块是逻辑的主载体，个股从属于板块逻辑
7. A股特殊微结构约束：T+1交易制度（买入后次日才能卖）、涨跌停限制（±10%/±20%）、印花税 — 这些影响"能否执行"逻辑判断后的操作，系统需提醒

## A股微结构约束

系统需要知晓以下约束，以便正确给出操作建议:

| 约束 | 影响 |
|------|------|
| T+1 | 买入后当日无法卖出，系统给出的操作建议必须考虑隔日风险 |
| 涨跌停 ±10% | 主板涨跌停限制，创业板/科创板 ±20% — 逻辑确认后可能出现连续涨停无法买入 |
| 集合竞价 | 9:15-9:25 集合竞价期间信号价值高，但不可执行 |
| 停牌 | 个股停牌期间无法交易，逻辑变更需要事后确认 |

## The Trading Logic Lifecycle

识别一个完整周期：

1. **发现逻辑** — AI综合新闻/研报/政策/供应链数据，识别当前主导逻辑
2. **追踪强度** — 每日计算逻辑强度占比（主导/次要权重）
3. **监控拐点** — 检测关键信号（政策转向、成本突破、量价异动等）
4. **确认切换** — 新逻辑确认（右侧）或前兆确认（左侧）
5. **输出决策** — 板块K线标注逻辑区间 + 个股服从板块的具体表现差异

## Premises

1. 每段行情由主导逻辑驱动，噪音不改方向 — ✅ agreed
2. 主导逻辑可被系统化识别和追踪（不一定立即发现，但趋势确立后需尽快识别）— ✅ agreed with nuance
3. 逻辑转变的前兆信号可追踪（黑天鹅例外，但大逻辑影响长/大的趋势段）— ✅ agreed with qualification
4. A股量化放大散户情绪差，认知差可被系统化利用 — ✅ agreed

## Approaches Considered

### Approach A: Regime Logic Tracker (MVP)
Build on DSA's existing pipeline. Daily sector logic extraction via AI. K-line overlay in web UI. Start with 2-3 sectors.
- **Effort**: S | **Risk**: Low
- **Pros**: Ships in days, proves concept, reuses DSA data/LLM pipeline
- **Cons**: Limited data breadth, tied to DSA architecture
- **Reuses**: DSA data_provider, LLM analysis, web dashboard

### Approach B: AI Researcher Agent
Autonomous multi-agent researchers with specialized web crawlers, report parsers, policy monitors.
- **Effort**: XL | **Risk**: High
- **Pros**: Highest ceiling, fully automated, broadest data coverage
- **Cons**: Complex crawler infrastructure, high dev time, maintenance burden
- **Reuses**: None — ground-up build

### Approach C: Hybrid Architecture (RECOMMENDED)
Standalone sector logic registry + logic synthesis engine as core. DSA reuses its data pipeline. Dashboard + alerts + deep reports as output. Multi-source data collectors feeding central engine.
- **Effort**: M | **Risk**: Medium
- **Pros**: Clean architecture, scales independently, DSA gets enhanced analytics, sharable output
- **Cons**: More upfront design than MVP, need to build the logic registry abstraction
- **Reuses**: DSA data_provider (partial), web dashboard UI, LLM channels

## Recommended Approach: Hybrid Architecture

Build the **Sector Logic Engine (SLE)** as a standalone core, with DSA as a consumer and contributor.

### Key Data Structures

**Trading Logic (交易逻辑):**
```json
{
  "sector": "CPO/光通信模块",
  "sector_code": "880792",
  "logic_id": "logic_2026_04_cpo_ai_demand",
  "title": "AI算力基础设施需求爆发",
  "description": "海外云厂商资本开支持续高增长，国内算力规划落地，CPO作为光通信核心环节直接受益于算力需求扩张",
  "category": "产业趋势",  // 产业趋势 | 政策驱动 | 成本反转 | 流动性 | 事件驱动
  "identified_date": "2026-03-15",

  "confidence": 0.85,
  // confidence = 逻辑本身的可信度。由两部分加权:
  //   evidence_quality (0-1): 证据源的权威性 × 交叉验证程度（单源0.3, 双源0.6, 三源+0.8+）
  //   price_alignment (0-1): 逻辑与价格走势的一致性（板块价格与逻辑方向一致程度）
  //   confidence = 0.6 * evidence_quality + 0.4 * price_alignment
  //   仅当 confidence >= 0.5 时才标记为 dominant

  "strength": 0.72,
  // strength = 当前时刻该逻辑在市场中占多大比重。由合成引擎 LLM 给定归一化权重:
  //   输入: 该板块所有证据源 + 近7日股价走势
  //   输出: sum(strength of all active logics) = 1.0
  //   strength_trend: rolling 3日强度变化方向 (increasing | stable | declining)
  "strength_trend": "increasing",

  "evidence_sources": [
    {"source": "news_article_123", "weight": 0.3, "summary": "三大云厂商Q1资本开支同比+40%", "credibility": "high"},
    {"source": "research_report_456", "weight": 0.4, "summary": "券商深度报告：AI算力链景气度分析", "credibility": "high"},
    {"source": "supply_chain_signal", "weight": 0.3, "summary": "中际旭创订单排满至Q3", "credibility": "medium"}
  ],
  "status": "dominant"  // dominant | secondary | emerging | fading | dead
}
```

**Data Quality Handling:**
- 矛盾数据（同一板块两份报告说相反逻辑）: 标记 `confidence = MIN(confidence, 0.4)`，等待第三源确认
- 证据源权威性分层: 官方文件/券商报告 > 主流财经媒体 > 自媒体/论坛 > 匿名/无来源
- 单一低可信度源不足以建立 dominant logic（需至少一个高可信度源 + 一个中可信度源交叉验证）
- AI 合成失败（LLM 返回空/乱码/无法提取）: 降级为 `status = "unknown"`，dashboard 显示 "当前板块逻辑不明确 — 不建议基于该板块盲目交易"

### LLM Synthesis Validation & Fallback

The LLM synthesis engine is the highest-risk component. It must defend itself:

1. **Schema validation**: Every LLM output is validated against Pydantic models. If parse fails → retry once → if still fails → status = "unknown"
2. **Evidence check**: If LLM claims a logic but produces zero evidence sources → confidence = 0.2 → status = "unknown"
3. **Cross-run consistency**: Run synthesis 2x with slightly different prompts on different days. If results match → confidence += 0.1. If they disagree → confidence -= 0.2 → flag to user "逻辑存在分歧"
4. **Price validation**: After LLM identifies dominant logic, check if sector price trend aligns (using DSA's existing price data). If price direction contradicts logic for >2 consecutive days → confidence drops and flip signal fires
5. **Manual override**: User can always override any AI-generated logic with their own assessment (manual entry with `source = "human_override"`, credibility = user-defined)
6. **Fallback chain**: LLM success → structured JSON. LLM fail → retry once → still fail → status = "unknown", dashboard shows "当前板块逻辑不明确"

**Phase 4 — DSA Integration (~1 week)**
- DSA analysis pipeline consumes logic data — adds logic context to individual stock analysis

### LLM Prompt Strategy

The synthesis engine uses a two-stage prompt:

**Stage 1 — Logic Extraction:**
Given all articles/reports/data for sector X in date range [T-7, T]:
1. What is the CURRENT dominant trading logic?
2. What evidence supports it (ranked by credibility)?
3. What are the secondary logics?
4. Are there any emerging or fading signals?

**Stage 2 — Logic Strength Calculation:**
Given N logics for sector X:
1. Calculate relative strength based on: evidence volume, source credibility, price action confirmation
2. Output: logic ID → strength score (0-1), trend direction

### Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|------------|
| LLM hallucination (inventing plausible logics) | High | High | Evidence chain required (every logic must cite specific sources). Schema validation. Price alignment check. |
| Over-confident flip signals | Medium | High | Confidence scoring with explicit methodology. Flip requires 2+ conditions met. Left-side signals flagged as "信号确认中" not "逻辑已切换". |
| Data pipeline fragility | High | Medium | Each collector independently fault-tolerant. Single collector failure does not stop synthesis. Dashboard shows data source health. |
| Black swan events | Low | High | System flags "unknown regime" when confidence < 0.3. "看不懂不盲做" guardrail enforced. |
| Confidence score meaningless | Medium | Medium | Now has explicit formula: `0.6*证据质量 + 0.4*价格对齐度`. Backtest against known flips to calibrate weights. |
| Strength calculation produces noise | Medium | High | LLM outputs normalized weights (sum=1.0) across all active logics for a sector. Rolling 3d smoothing. Manual override available. |

The system does NOT give entry/exit prices. It provides the cognitive framework — user still manages position sizing and timing. A股 T+1 约束意味着即使当日确认逻辑切换，也要隔日才能执行退出。
